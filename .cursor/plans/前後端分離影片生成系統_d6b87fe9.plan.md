### 主要 API 端點

1. **POST /api/upload** - 上傳 PDF 和簡報大綱

- 接收：multipart/form-data (pdf_file, abstract_file)
- 返回：task_id, status

2. **POST /api/generate-script** - 生成腳本

- 接收：task_id, length_mode (SHORT/MEDIUM/LONG)
- 調用 Gemini 3 Flash 生成腳本
- 返回：script_content, transcription_data

3. **GET /api/task/{task_id}/status** - 查詢任務狀態

- 返回：status, progress, current_step

4. **GET /api/task/{task_id}/script** - 獲取生成的腳本

- 返回：script_content (可編輯)

5. **PUT /api/task/{task_id}/script** - 更新腳本

- 接收：edited_script
- 返回：updated_script

6. **POST /api/task/{task_id}/generate-video** - 開始生成影片

- 接收：video_params (fps, resolution, etc.)
- 啟動背景任務
- 返回：task_id

7. **GET /api/task/{task_id}/video** - 下載影片

- 返回：video file (streaming)

8. **WebSocket /ws/task/{task_id}** - 即時進度更新

- 推送：progress updates, status changes

### 核心服務

**gemini_service.py**

- 整合 Google Gemini 3 Flash API
- 使用 `prompt.py` 中的 `SOLO_PRESENTATION_PROMPT` 和 `TRANSCRIPT_REWRITER_PROMPT`
- 生成腳本並轉換為 transcription 格式

**video_generator.py**

- 整合 `test.py` 中的函數：
- `pdf_to_images()`
- `process_transcription_to_audio()`
- `create_video()`
- 支援背景任務執行
- 進度回報機制

**tasks.py**

- 使用 Celery 或 asyncio 背景任務
- 任務狀態管理
- 進度追蹤

## 資料流程

1. **上傳階段**

- 使用者上傳 PDF + 簡報大綱
- 後端儲存檔案並建立任務

2. **腳本生成階段**

- 後端讀取簡報大綱和 PDF
- 調用 Gemini 3 Flash 使用 `SOLO_PRESENTATION_PROMPT`
- 生成初步腳本
- 使用 `TRANSCRIPT_REWRITER_PROMPT` 轉換格式
- 返回可編輯腳本給前端

3. **腳本編輯階段**

- 使用者在前端編輯腳本
- 儲存更新到後端

4. **影片生成階段**

- 使用者確認參數並啟動生成
- 後端啟動背景任務
- 執行 `test.py` 中的 `create_video()` 函數
- 透過 WebSocket 推送進度

5. **下載階段**

- 生成完成後提供下載連結
